**Status: Implemented** -- C runtime backing.

The `std.i18n` module provides string externalisation and internationalisation support. Toke source code stays lightweight and focused on logic -- all user-facing text (labels, messages, error strings) is stored in external data files that can be swapped per locale at runtime.

String bundles can be written in any of toke's supported serialization formats: TOON (default), YAML, or JSON. The module automatically detects the format and resolves locale-specific files with a fallback chain.

## Design Philosophy

Toke code generators focus on generating **code**, not strings. Variable text is externalised into data files that existing LLM models and tokenisers handle well. This keeps toke source compact and makes internationalisation a core capability rather than an afterthought.

## Types

### $i18nbundle

A loaded string bundle containing key-value pairs for a specific locale.

### $i18nerr

A sum type representing i18n operation failures.

| Variant | Field Type | Meaning |
|---------|------------|---------|
| $notfound | $str | No bundle file found for the given base path and locale |
| $parse | $str | The bundle file exists but could not be parsed |

## Functions

### i18n.load(base: $str; locale: $str): $i18nbundle!$i18nerr

Loads a string bundle for the given locale. Searches for files in this order:

1. `{base}.{locale}.toon`
2. `{base}.{locale}.yaml`
3. `{base}.{locale}.json`
4. `{base}.toon` (fallback)
5. `{base}.yaml`
6. `{base}.json`

Returns `$i18nerr.$notfound` if no matching file is found.

```toke
m=app;
i=i18n:std.i18n;
f=main():i64{
  let ui=i18n.load("lang/strings";"fr")|{$ok:b b;$err:e i18n.empty()};
  < 0
};
```

### i18n.str(bundle: $i18nbundle; key: $str): $str

Looks up a string by key. If the key is not found, returns the key itself as a safe fallback (no crash, always produces visible output).

```toke
m=app;
i=i18n:std.i18n;
f=main():i64{
  let ui=i18n.load("lang/ui";"en")|{$ok:u u;$err:e i18n.empty()};
  let greeting=i18n.str(ui;"greeting");
  < 0
};
```

### i18n.fmt(bundle: $i18nbundle; key: $str; args: $str): $str

Looks up a key and substitutes `{placeholder}` tokens with values from the args string. Args are pipe-delimited `name=value` pairs.

```toke
m=app;
i=i18n:std.i18n;
f=main():i64{
  let ui=i18n.load("lang/ui";"en")|{$ok:u u;$err:e i18n.empty()};
  let msg=i18n.fmt(ui;"welcome";"name=Alice|count=5");
  < 0
};
```

### i18n.locale(): $str

Returns the system locale string from the `LC_ALL` or `LANG` environment variable, with the encoding suffix stripped (e.g., `en_AU` from `en_AU.UTF-8`).

```toke
m=app;
i=i18n:std.i18n;
f=main():i64{
  let loc=i18n.locale();
  < 0
};
```

## Bundle File Formats

String bundles can be written in any supported format:

### TOON (recommended)

```
strings[3]{key,value}:
greeting|Hello
farewell|Goodbye
welcome|Welcome, {name}!
```

### YAML

```yaml
greeting: Hello
farewell: Goodbye
welcome: "Welcome, {name}!"
```

### JSON

```json
{"greeting":"Hello","farewell":"Goodbye","welcome":"Welcome, {name}!"}
```

## Usage Examples

```toke
m=app;
i=i18n:std.i18n;
i=log:std.log;

f=main():i64{
  let loc=i18n.locale();
  let ui=i18n.load("lang/ui";loc)|{$ok:u u;$err:e i18n.empty()};
  let title=i18n.str(ui;"apptitle");
  let welcome=i18n.fmt(ui;"welcomemsg";"user=Alice|role=admin");
  log.info(title;@());
  log.info(welcome;@());
  <0;
};
```
