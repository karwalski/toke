# std.i18n -- Internationalisation

## Overview

The `std.i18n` module loads locale-specific string bundles from disk and
retrieves translated messages by key. Locale files are plain text files where
each line has the form `key=value`. A directory of locale files (one file per
locale, named `<locale>.txt`) is loaded at startup and queried at runtime.

The module exposes four functions and two types. `i18n.load` reads a locale
directory and returns an `I18nBundle` or an `I18nErr`. The remaining functions
operate on a loaded bundle to fetch and format translated strings.

## Types

### I18nBundle

An opaque handle representing a loaded set of key-value translations for a
single locale. Obtained from `i18n.load` and passed to `i18n.get` and
`i18n.fmt`.

### I18nErr

Error union returned by `i18n.load` when loading fails.

| Variant    | Meaning                                              |
|------------|------------------------------------------------------|
| `NotFound` | The locale directory or requested locale file does not exist. |
| `Parse`    | A locale file was found but contains malformed lines. |

## Functions

### i18n.load(dir: str; locale: str): I18nBundle!I18nErr

Reads the locale file at `<dir>/<locale>.txt` and returns an `I18nBundle`
containing every key-value pair found. Each line in the file must be in the
format `key=value`. Blank lines and lines that do not contain `=` produce a
`Parse` error.

Returns `I18nErr.NotFound` if the directory or locale file does not exist.
Returns `I18nErr.Parse` if any line is malformed.

**Example — locale directory layout:**

```
locales/
  en.txt
  fr.txt
  ja.txt
```

**Example — en.txt:**

```
greeting=Hello
farewell=Goodbye
item_count=You have {0} items
```

**Example:**

```toke
let bundle = i18n.load("locales"; "en");
(* bundle is I18nBundle on success, I18nErr on failure *)
```

**Error handling:**

```toke
let result = i18n.load("locales"; "zz");
(* result is I18nErr.NotFound when zz.txt does not exist *)
```

### i18n.get(bundle: I18nBundle; key: str): str

Looks up `key` in `bundle` and returns the associated translated string. If
the key is not present the function returns the key itself, making missing
translations visible in output without crashing.

**Example:**

```toke
let bundle = i18n.load("locales"; "en");
let msg = i18n.get(bundle; "greeting");
(* msg == "Hello" *)

let missing = i18n.get(bundle; "no_such_key");
(* missing == "no_such_key" — key echoed back *)
```

### i18n.fmt(bundle: I18nBundle; key: str; args: str): str

Looks up `key` in `bundle`, then substitutes positional placeholders `{0}`,
`{1}`, ... with the corresponding values from `args`. The `args` string is a
semicolon-separated list of replacement values. If the key is missing, the key
itself is returned without substitution.

Placeholders that exceed the number of supplied arguments are left in the
output unchanged, making formatting mismatches easy to spot.

**Example:**

```toke
let bundle = i18n.load("locales"; "en");

(* en.txt contains: item_count=You have {0} items *)
let msg = i18n.fmt(bundle; "item_count"; "5");
(* msg == "You have 5 items" *)
```

**Multiple arguments:**

```toke
(* en.txt contains: welcome=Hello {0}, you have {1} messages *)
let msg = i18n.fmt(bundle; "welcome"; "Alice;3");
(* msg == "Hello Alice, you have 3 messages" *)
```

### i18n.locale(): str

Returns the locale string of the current runtime environment (e.g. `"en"`,
`"fr"`, `"ja"`). This value is derived from the host operating system locale
settings and can be passed directly to `i18n.load` as the `locale` argument.

**Example:**

```toke
let loc = i18n.locale();
let bundle = i18n.load("locales"; loc);
let msg = i18n.get(bundle; "greeting");
```

## Usage Pattern -- Locale-File Loading and Translation

A typical application loads a bundle once at startup using the detected locale,
then passes the bundle through the call graph for key-based lookups.

```toke
m=myapp;
i=i18n:std.i18n;

f=main():void{
  let loc = i18n.locale();
  let bundle = i18n.load("locales"; loc);

  (* Simple lookup *)
  let title = i18n.get(bundle; "app_title");

  (* Formatted lookup with arguments *)
  let status = i18n.fmt(bundle; "status_line"; "42;OK");

  (* Pass bundle to other functions *)
  render_ui(bundle);
};

f=render_ui(b: I18nBundle): void{
  let label = i18n.get(b; "submit_button");
  (* use label *)
};
```

## Notes

- Locale files are plain UTF-8 text. There is no nested structure; each key
  maps to a single string value.
- Keys should use `snake_case` by convention so they are easy to grep across
  both code and locale files.
- `i18n.get` never fails at runtime. Missing keys return the key string,
  keeping the application running while making gaps visible.
- `i18n.fmt` uses `{0}`, `{1}`, ... as positional placeholders. The `args`
  parameter is a single string with values separated by semicolons.
- To support a new locale, add a `<locale>.txt` file to the locale directory.
  No code changes are required.
